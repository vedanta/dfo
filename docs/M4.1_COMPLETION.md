### 2. Analysis Modules (✅ Complete)

**Low-CPU Analysis (`src/dfo/analyze/low_cpu.py`)**
- Identifies VMs with CPU <20% over 14 days
- Recommends smaller SKU in same family (e.g., D4→D2, E8→E4)
- Handles special cases (B-series can go to 1 vCPU)
- Calculates potential savings from downsizing
- 30 comprehensive unit tests

**Stopped VMs Analysis (`src/dfo/analyze/stopped_vms.py`)**
- Identifies VMs stopped/deallocated for 30+ days
- Estimates disk storage costs (~10% of VM cost)
- Recommends Delete (>90 days) or Review (30-90 days)
- 19 comprehensive unit tests

## Key Technical Achievements

1. **Dynamic CLI Architecture**: The CLI now dynamically loads and calls analysis modules based on rules, eliminating hardcoded function names

2. **Consistent Analysis Pattern**: All 3 analyses follow the same structure:
   - Analyze function: `analyze_{type}_vms()`
   - Query function: `get_{type}_vms()`
   - Summary function: `get_{type}_vm_summary()` or `get_{type}_summary()`

3. **SKU Downsizing Logic**: Intelligent VM SKU recommendations that:
   - Halve vCPU count while preserving series/modifiers/generation
   - Handle special cases (B-series minimum 1 vCPU)
   - Maintain consistent naming patterns

4. **Severity Classification**: Unified severity levels across all analyses:
   - Critical: ≥$500/month savings
   - High: $200-499/month
   - Medium: $50-199/month
   - Low: <$50/month
