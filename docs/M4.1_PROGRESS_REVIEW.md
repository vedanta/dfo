### 2. Low-CPU Analysis Module (✅ COMPLETE)
**Planned:** `src/dfo/analyze/low_cpu.py` with SKU recommendations
**Completed:**

**Module:** `src/dfo/analyze/low_cpu.py` (598 lines)

**Key Functions:**
- `analyze_low_cpu_vms()` - Main analysis entry point
- `_analyze_vm_cpu_for_rightsizing()` - CPU analysis logic
- `_recommend_smaller_sku()` - SKU downsizing (D4→D2, E8→E4, etc.)
- `_parse_sku()` - Parse VM SKU components
- `_determine_severity()` - Severity classification
- `get_low_cpu_vms()` - Query results with filters
- `get_low_cpu_summary()` - Summary statistics

**Features:**
- ✅ Identifies VMs with CPU < 20% over 14 days
- ✅ Recommends smaller SKU in same family (halves vCPU count)
- ✅ Preserves series, modifiers, and generation
- ✅ Minimum size: 2 vCPU (1 for B-series)
- ✅ Calculates actual costs using Azure pricing
- ✅ Computes savings amount and percentage
- ✅ Determines severity: Critical/High/Medium/Low

**SKU Logic Examples:**
```
Standard_D4s_v5 → Standard_D2s_v5   (4 vCPU → 2 vCPU)
Standard_E8s_v5 → Standard_E4s_v5   (8 vCPU → 4 vCPU)
Standard_B4ms → Standard_B2ms       (4 vCPU → 2 vCPU)
Standard_D2s_v5 → None              (already minimum)
```
